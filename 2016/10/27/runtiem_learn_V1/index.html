#runtime详解

官方文档并配合代码详解runtime方法的作用

###方法

runtime方法分为多种,将一一进行详解

####Working with Classes(使用类)

  1. class_getName

    Returns the name of a class.(返回类名)

  2. class_getSuperclass(获取父类)

    Returns the superclass of a class.(返回父类名称)

  3. class_setSuperclass(重新给一个类设置父类,已过时)

    Sets the superclass of a given class.

  4. class_isMetaClass

    Returns a Boolean value that indicates whether a class object is a metaclass.(返回一个对象是否为元类(什么是元类,参考))

    • 每一个类里面都有一个isa指针,指向它的元类,每个类的元类都是独一无二而.
    • 元类里面存储了一个类的类方法,当调用一个类方法的时候,会在该类的metaClass里的methodList进行寻找.
    • 每一个 Meta Class 的 isa 指针都指向最上层的 Meta Class，即 NSObject 的 MetaClass，而最上层的 MetaClass 的 isa 指针又指向自己(参考了上方链接)

  5. class_getInstanceSize

    Returns the size of instances of a class.(返回一个类的实例大小)

  6. class_getInstanceVariable

    参数: 类,类的实例变量名称,以下划线开头

    返回值:Returns the Ivar for a specified instance variable of a given class.(变量信息的16进制内存地址)

  7. class_getClassVariable

    参数:类名

    返回值:Returns the Ivar for a specified class variable of a given class.

    未清楚其含义,stackflow上找到如下解释:class_getClassVariable(cls, name) merely calls class_getInstanceVariable(cls->isa, name).,中文意思就是:class_getClassVariable只调用class_getInstanceVariable。

  1. class_addIvar(添加实例变量)

    此功能只能objc_allocateClassPair后objc_registerClassPair之前被调用。添加一个实例变量，以现有的类不被支持。
